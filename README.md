＜ターン制バトルシステム (Turn-based Battle System)＞  
Pythonの基礎的な制御構文とオブジェクト指向（クラス）の概念を活用して制作した、コマンド選択式のバトルシミュレーターです。 
  
＜プロジェクト概要＞  
大学のプログラミング演習の一環として、「スタミナ」というリソース管理の概念を取り入れた戦闘システムを構築しました。 単なるHPの削り合いではなく、スタミナを考慮した戦略的な行動選択が必要なゲームデザインを目指しました。
  
🛠 技術スタック  
言語: Python 3.x  
概念: オブジェクト指向プログラミング（クラス・メソッド）、ループ処理、条件分岐   

主な機能とこだわり  
1. クラスによるキャラクター管理charactorクラスを定義し、プレイヤーと敵キャラクターの属性（HP、攻撃力、防御力、回避率、スタミナ等）を一括管理しています。 これにより、異なるステータスを持つ「戦士」「魔法使い」「アーチャー」といった複数の職業を容易に実装できるようにしました。
2. スタミナ連動型のアクションシステム各行動（攻撃・防御・回避）にスタミナの増減を設定し、戦況に応じたリソース管理を組み込みました。   攻撃: スタミナを消費。不足している場合は行動が失敗し、スタミナの微回復に留まる。   防御・回避: スタミナを回復させつつ、受けるダメージの軽減や回避を試みる。
3. 安全なダメージ計算ダメージ計算時に max(0, attack - guard) を用いることで、防御力が攻撃力を上回った際にHPが回復してしまうといったバグを防止し、計算の堅牢性を高めました。

開発を通じて学んだこと  
・条件分岐の複雑化への対応: battleAction メソッド内での戻り値を利用したターン制御など、複雑なロジックを整理して実装する難しさと重要性を学びました。  
・ゲームループの実装: while True ループ内での生死判定と、適切なタイミングでのループ脱出（break）の処理を習得しました。
